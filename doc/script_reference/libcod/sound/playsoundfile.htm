playSoundFile( <sound index>, <source> )

Module: Sound

Summary:

Plays a song via voice chat, based on a sound index returned from loadSoundFile. Uses the voice channel, therefore only one sound can be played at a time. A script notify with string sound_file_done is emit as soon as the last voice packet has been queued for sending. Note that actual voice chat coming from the same source player will interfere with the audio stream. This function also works with sv_voice set to "0". Playback stops if the source player disconnects. Playback does not stop if mutePlayer is issued in the client console. Playback stops if cl_voice is set to "0" on the client or if the source is muted via uiScript.

Note: Due to engine limitations, the sound quality is rather low and also depends on the connection quality (packet loss) of the receiving player.



Call this on:

The player



Example:

self playSoundFile(index, self getEntityNumber());



Required Args:

  • 1 : <sound index> A sound index returned from loadSoundFile.


Optional Args

  • 1 : <source> The entity number of the emitting player (default: self).
