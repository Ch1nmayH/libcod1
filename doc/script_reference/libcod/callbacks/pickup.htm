Pickup

Module: Callbacks

Summary:

Calls CodeCallback_Pickup() in maps/mp/gametypes/_callbacksetup.gsc if a weapon, ammo, or health is picked up by a player. For ammo and health, only the first parameter is defined. The item/entity is removed automatically.

Note: The existance of the callback function is checked only once on server start. Functions added later are not taken into account, even after a map switch.



Called on:

The player



Parameters:

  • 1 : <type>: The type of the item that was picked up (either "ammo", "weapon", or "health").
  • 2 : <weapon>: Weapon file name of the picked up weapon.
  • 3 : <name>: The weapon name as displayed in the player hud.
  • 4 : <quantity>: The ammo count of the picked up weapon.
  • 5 : <origin>: The origin of the picked up weapon.
  • 6 : <angles>: The angles of the picked up weapon.


Example:

CodeCallback_Pickup(type, weapon, name, quantity, origin, angles)
{
    if(type == "weapon")
        self setWeaponSlotWeapon("primary", weapon);
}
