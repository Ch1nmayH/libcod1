UserInfo Updates

Module: Callbacks

Summary:

Calls CodeCallback_UserInfoChanged() in maps/mp/gametypes/_callbacksetup.gsc when the value of a client-sided userinfo dvar should be updated. Requires processClientUserinfoChange to be called to make the server continue processing the update.

Note: The existance of the callback function is checked only once on server start. Functions added later are not taken into account, even after a map switch.

Note: Avoid using wait statements (unless threaded) in this callback before calling processClientUserinfoChange.



Called on:

The player



Example:

CodeCallback_UserInfoChanged()
{
    oldname = self.name;
    newname = self getUserinfo("name");
    // Add custom logic on rename (for example)
    self processClientUserinfoChange();
}
