Errors

Module: Callbacks

Summary:

Calls CodeCallback_Error() in maps/mp/gametypes/_callbacksetup.gsc. Called immideately on critical/terminal script runtime errors, else on the next invocation of G_RunFrame. Not called on script compile errors.

Note: The existance of the callback function is checked only once on server start. Functions added later are not taken into account, even after a map switch.

Note: Avoid using wait statements in this callback.



Parameters:

  • 1 : <critical> Boolean flag indicating whether the error is critical/terminal, thus potentially resulting in a server shutdown.
  • 2 : <internal function> Name of the internal function from where the error was raised (e.g., Com_Error).
  • 3 : <message> Error message (this does not include details like the exact position within the script code).


Example:

CodeCallback_Error(critical, internal_function, message)
{
    if(critical || internal_function == "Com_Error") {
        logPrintConsole("Aight imma head out!\n");
    } else {
        // If developer mode is on, check console_mp_server.log for details
        iprintLn("error: " + message);
    }
}
